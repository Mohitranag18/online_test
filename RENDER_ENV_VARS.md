# Environment Variables for Render Deployment

## Required Environment Variables

These variables need to be set in Render Dashboard → Your Web Service → Environment tab:

### Django Settings
```
SECRET_KEY=<auto-generated-by-render>
DEBUG=False
ALLOWED_HOSTS=yaksh-backend.onrender.com
DOMAIN_HOST=https://yaksh-backend.onrender.com
```

### Database & Redis
```
DATABASE_URL=postgresql://neondb_owner:npg_9HAJz7WwSEiC@ep-long-tree-ad7bfusc-pooler.c-2.us-east-1.aws.neon.tech/neondb?sslmode=require
REDIS_URL=<auto-provided-by-render-from-blueprint>
```

**⚠️ Important:** You need to manually add the `DATABASE_URL` from your Neon DB connection string!

### CORS for Frontend
```
CORS_ALLOWED_ORIGINS=https://your-frontend.vercel.app
```

**Important:** Replace `your-frontend.vercel.app` with your actual Vercel frontend URL.

## How to Set Environment Variables in Render

1. Go to [Render Dashboard](https://dashboard.render.com)
2. Click on your web service (`yaksh-backend`)
3. Go to **Environment** tab
4. Click **Add Environment Variable**
5. Add each variable from above
6. Click **Save Changes** (this will trigger a redeploy)

---

## Auto-Generated Variables

These are created automatically by the `render.yaml` blueprint:

- `SECRET_KEY` - Generated by Render
- `REDIS_URL` - Linked from Redis service
- `DEBUG` - Set to False
- `PYTHON_VERSION` - Set to 3.9.0

You MUST manually add these after deployment:
- `DATABASE_URL` - Your Neon DB connection string (provided above)
- `ALLOWED_HOSTS` - Your Render service URL
- `DOMAIN_HOST` - Your full domain with https://
- `CORS_ALLOWED_ORIGINS` - Your Vercel frontend URL

**Note:** Since you're using Neon DB instead of Render PostgreSQL, you need to add the DATABASE_URL manually!
